# Code Review: Personal Finance Docusaurus

## 🔴 Critical Issues

### 1. **Generated PDF Files Not in .gitignore** 
- **Severity:** HIGH
- **Issue:** `*.pdf.mdx` files are auto-generated by `pdf-pages-plugin.js` but NOT listed in `.gitignore`
- **Impact:** These files will be committed to git, bloating the repository and causing merge conflicts
- **Fix:** Add to `.gitignore`:
  ```
  # Generated PDF pages
  **/*.pdf.mdx
  ```

### 2. **Unsafe dotenv Plugin Configuration**
- **Severity:** HIGH  
- **Location:** `docusaurus.config.ts` line 43
- **Issue:** `safe: false, systemvars: true` exposes all system environment variables to the build
- **Risk:** Credentials and secrets in env vars could be leaked in the built site
- **Fix:** 
  ```typescript
  {
    path: './.env',
    safe: true,        // Validate against .env.example
    systemvars: false, // Don't expose system vars
    silent: false,
  }
  ```

### 3. **Unencrypted Decap CMS Script**
- **Severity:** MEDIUM-HIGH
- **Location:** `docusaurus.config.ts` line 83
- **Issue:** Loading Decap CMS from unpkg CDN without integrity checking or fallback
- **Risk:** Third-party script injection vulnerability; will break if CDN is down
- **Fix:** Use a specific version and consider self-hosting:
  ```typescript
  src: 'https://unpkg.com/decap-cms@3.0.0/dist/decap-cms.js', // Pin version
  crossOrigin: 'anonymous',
  integrity: 'sha384-...', // Add subresource integrity
  ```

- **Severity:** MEDIUM
- **Location:** `docusaurus.config.ts` lines 109-130
- **Issue:** Navbar links use `/Accounting/`, `/Banks/` but one old commented config uses `docSidebar` type
- **Problem:** Mixed navigation paradigm - some links bypass sidebar
- **Fix:** Choose one approach:

### 6. **PDF Viewer Hardcoded Height**
- **Severity:** LOW-MEDIUM
- **Location:** `src/components/PDFViewer.module.css` line 42
- **Issue:** Fixed `height: 600px` doesn't account for mobile responsiveness
- **Fix:** Add mobile breakpoint:
  ```css

    width: 100%;
    height: 600px;
    background-color: #f5f5f5;
  }

    }
  }
  ```

### 7. **PDFViewer Missing Error Handling**
- **Severity:** MEDIUM
- **Location:** `src/components/PDFViewer.tsx`
- **Issue:** No fallback if iframe fails to load PDF
- **Fix:** Add error handling:
  ```typescript
  <iframe
    src={src}
    title={title}
    className={styles.pdfFrame}
    frameBorder="0"
    onError={() => console.error(`Failed to load PDF: ${title}`)}
  />
  <p style={{marginTop: '1rem', color: 'var(--ifm-color-danger)'}}>
    If PDF doesn't load, try downloading it using the button above.
  </p>
  ```

### 9. **Search Plugin Might Index Generated PDFs**
- **Severity:** LOW-MEDIUM
- **Location:** `docusaurus.config.ts` line 49-54
- **Issue:** Search plugin doesn't exclude auto-generated `.pdf.mdx` files
- **Impact:** Users might see duplicate/confusing search results
- **Fix:** Add exclusion pattern:
  ```typescript
  [
    '@easyops-cn/docusaurus-search-local',
    {
      hashed: true,
      language: ['en'],
      highlightSearchTermsOnTargetPage: true,
      explicitSearchResultPath: true,
      exclude: ['**/*.pdf.mdx'], // Add this
    },
  ],
  ```

### 10. **Missing TypeScript Strict Mode**
- **Severity:** LOW
- **Location:** `tsconfig.json`
- **Issue:** No `strict: true` setting for better type safety
- **Fix:** Consider enabling:
  ```json
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true
  }
  ```
